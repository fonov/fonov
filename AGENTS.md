# AGENTS.md

## Cursor Cloud specific instructions

### Overview
Single product: **FonovTest** — a client-only React SPA (Create React App, `react-scripts` 1.0.17)
for diagnosing a used iPhone before purchase. The only runnable service lives in `fonov-app/`.
There is no backend, database, or other service. Other top-level dirs (`source/`, `trans/`, `ui/`)
are non-code assets/tooling.

### Node version (important)
This is a legacy stack that only builds on **Node 10** (installed via `nvm`). Newer Node breaks the
old webpack/uglify toolchain. The interactive shell is preconfigured (via `~/.bashrc`) to select
Node 10, but note that a system `node` at `/exec-daemon/node` (v22) may shadow it in some shells —
`npm` still resolves to the Node 10 npm, which is what matters for install/build/run. If `node -v`
does not report v10, run: `nvm use 10 && export PATH="$HOME/.nvm/versions/node/v10.24.1/bin:$PATH"`.

### Generated file: `maps.json` (required to compile)
`fonov-app/src/actions/image-manager.js` imports `src/assets/img/maps.json`, which is a **generated**
file (missing on a clean checkout). Without it, both `npm start` and the tests fail to compile.
It is generated by the committed script `src/assets/img/_maps.js` (uses the `image-size` dep):
`cd fonov-app/src/assets/img && node _maps.js` (writes `maps.json` after ~1s).
The committed `maps.json` is generated on Linux; note the asset dirs contain both `Appearance/` and
`appearance/` (a macOS case-insensitivity artifact) — the generator lowercases map keys but keeps the
real-case `src` paths, so regenerate on Linux if paths ever fail to resolve.

### Dependency resolution (`react` / `createContext`)
`package.json` declares `"react": "^16.2.0"`. The repo's original `package-lock.json` was stale/incomplete
(it pinned `react@16.2.0` but was missing several declared deps such as `react-localize-redux` and
`react-vk`). Installing against it left `react` at 16.2.0 while newer transitive deps call
`React.createContext` (added in 16.3), which crashed the app at runtime with a `createContext` TypeError
(blank page). The lockfile has been regenerated so `react`/`react-dom` resolve to `16.14.0` (still within
`^16.2.0`) and the tree is internally consistent. Do not re-pin `react` back to 16.2.0.

### Run / build / test (from `fonov-app/`)
- Run dev server: `BROWSER=none npm start` (Create React App, serves on port 3000).
- Build: `npm run build`.
- Test: `npx jest --testURL=http://localhost`. The bare `npm test` (`jest`) fails with
  `SecurityError: localStorage is not available for opaque origins`; the `--testURL` flag fixes it.
  The single existing test (`src/__test__/actions/image-manager.test.js`) still **fails** on a
  pre-existing code/test mismatch (it passes `locale: {}` to `getActiveLanguage`, but
  `react-localize-redux@2.17.5` expects `locale.languages`). This is unrelated to environment setup.
